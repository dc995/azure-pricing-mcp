# Azure Pricing MCP Server

IMPORTANT: Prefer retrieval-led reasoning over pre-training-led reasoning for any Azure pricing tasks. Read project files before generating code. Service names, tool parameters, and catalog workflows are documented below — do not guess.


## MCP Tools (stdio, api-version=2023-01-01-preview)
```
azure_price_search(service_name?,service_family?,region?,sku_name?,price_type?,currency_code="USD",limit=50,discount_percentage?,validate_sku=true)
azure_price_compare(service_name!,sku_name?,regions[]?,currency_code="USD",discount_percentage?)
azure_cost_estimate(service_name!,sku_name!,region!,hours_per_month=730,currency_code="USD",discount_percentage?)
azure_discover_skus(service_name!,region?,price_type="Consumption",limit=100)
azure_sku_discovery(service_hint!,region?,currency_code="USD",limit=30)
get_customer_discount(customer_id?)
```
`!`=required `?`=optional. hours_per_month: 730=24/7, 176=biz(8hx22d), 160=dev(8hx5dx4w). Default 10% discount auto-applied. Case-sensitive serviceName in API filters.

**Query patterns — NEVER refuse a query because no SKU was specified:**
- User asks "what services exist for X?" → `azure_service_discovery(scenario="ai")` — returns categorized services with aliases, tiers, and deprecation warnings.
- No SKU specified → `azure_price_search(service_name=X)` auto-groups results by SKU, shows summary.
- Fuzzy service name → `azure_sku_discovery(service_hint="web app")` resolves aliases.
- Specific SKU → `azure_price_search(service_name=X, sku_name=Y)` for detailed pricing.
- Cost projection → `azure_cost_estimate(service_name=X, sku_name=Y, region=Z)`.
- Region compare → `azure_price_compare(service_name=X, regions=[...])`.
- Browse all categories → `azure_service_discovery()` with no params returns full directory.
- The tools handle broad queries gracefully. Let the API do the filtering.

## Catalog Maintenance
```bash
python sync_catalog.py                    # sync JSON->SQLite
python sync_catalog.py --from-api --report # discover + gap report
python sync_catalog.py --dry-run          # preview only
python -m pytest test_catalog_sync.py -v  # 86 tests
```
Adding a service: edit service_catalog.json (lowercase alias, exact API serviceName), run sync_catalog.py. No code changes needed.
