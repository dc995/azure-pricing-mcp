# Adding a New Azure Service — Standard Workflow

## Quick Reference

To add a new Azure service to the pricing MCP server, you only need to:

1. **Edit `service_catalog.json`** — add alias entries
2. **Run `python sync_catalog.py`** — sync to SQLite
3. **Done** — no code changes needed

---

## Step-by-Step

### 1. Find the Exact API Service Name

Query the Azure Retail Prices API to find the exact `serviceName`:

```bash
# Search by keyword
curl "https://prices.azure.com/api/retail/prices?api-version=2023-01-01-preview&\$filter=contains(serviceName,'YourService')" | python -m json.tool | grep serviceName

# Or check the discovered services in SQLite
python -c "import sqlite3; conn = sqlite3.connect('azure_services.db'); [print(r[0], '|', r[1]) for r in conn.execute('SELECT service_name, service_family FROM azure_services ORDER BY service_name')]"
```

> **Important**: The `serviceName` is case-sensitive in API filters. Use the exact name from the API.

### 2. Add Aliases to `service_catalog.json`

Open `service_catalog.json` and add entries to the `service_aliases` array:

```json
{"alias": "your common name",   "service_name": "Exact API Name", "category": "Category"},
{"alias": "abbreviation",       "service_name": "Exact API Name", "category": "Category"},
{"alias": "alternate term",     "service_name": "Exact API Name", "category": "Category"}
```

**Guidelines for aliases:**
- Use **lowercase** — the server lowercases all user input before matching
- Include the **full name**, **common abbreviation**, and **colloquial terms**
- Example: For "Azure Kubernetes Service" → `"kubernetes"`, `"aks"`, `"k8s"`, `"container service"`

### 3. Sync to SQLite

```bash
python sync_catalog.py
```

This will:
- Add new aliases to the database
- Update changed entries
- Prune aliases removed from JSON
- Log all changes

### 4. Verify

```bash
# Check the alias was added
python -c "import sqlite3; conn = sqlite3.connect('azure_services.db'); print(conn.execute(\"SELECT service_name FROM service_aliases WHERE alias='your-alias'\").fetchone())"
```

## Adding a Deprecated Service

Add to the `deprecated_services` array in `service_catalog.json`:

```json
{
  "alias": "old service name",
  "service_name": "Replacement API Service Name",
  "status": "retired|retiring|deprecated|rebranded",
  "retirement_date": "YYYY-MM-DD",
  "replacement": "Human-readable replacement name",
  "message": "User-facing warning message with migration guidance."
}
```

## Discovering New Services from the API

Run the sync utility with `--from-api` to scan for new services:

```bash
python sync_catalog.py --from-api --report
```

This queries every Azure service family and adds newly discovered services to the `azure_services` table. The `--report` flag shows which discovered services lack aliases.

## Architecture Summary

```
service_catalog.json  ──► sync_catalog.py ──► azure_services.db
(human-editable)          (CLI utility)       (runtime state store)
                                                     │
                                              azure_pricing_server.py
                                              (reads at startup via
                                               lazy-loaded properties)
```

- **Source of truth**: `service_catalog.json` (committed to git)
- **Runtime store**: `azure_services.db` (gitignored, regenerated by sync)
- **Server**: reads SQLite on first use, no hardcoded data
