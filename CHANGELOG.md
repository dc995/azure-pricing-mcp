# Changelog

All notable changes to the Azure Pricing MCP Server are documented here.

## [2.0.0] - 2026-03-20

### Fork by [dc995](https://github.com/dc995)

Forked from [charris-msft/azure-pricing-mcp](https://github.com/charris-msft/azure-pricing-mcp). This release is a major architectural change that externalizes all service data from code into a data-driven catalog.

### Added

- **Data-driven service catalog** (`service_catalog.json`) — 418 aliases mapping to 156 Azure API services. Edit JSON to add services; no code changes needed.
- **SQLite runtime store** (`azure_services.db`) — lazy-loaded by the server at startup, regenerated by sync utility.
- **`sync_catalog.py`** — CLI utility for JSON→SQLite sync with `--from-api` API discovery, `--report` gap analysis, `--dry-run` preview, prune/upsert/log.
- **`azure_service_discovery` tool** — new MCP tool for scenario-based service exploration ("what services exist for AI/data/security?"). Supports category browsing, keyword search, and full directory listing.
- **SKU tier mappings** — named SKU tiers (Fabric F2–F2048, Cosmos DB RU/s, AI Search tiers, OpenAI model pricing, Azure ML compute, AI Services) with computed pricing from base unit rates.
- **Data provenance** — every catalog entry tracks `source` URL and `source_date` for auditability.
- **Auto-summarize broad queries** — `azure_price_search` without a SKU filter now groups results by SKU instead of returning raw items or erroring.
- **`AGENTS.md`** — compressed passive context file (6.3 KB) following Vercel research showing AGENTS.md outperforms skills for agent accuracy (100% vs 53%).
- **Copilot CLI skill** (`azure-pricing-mcp.skill`) — packaged skill with inline alias index, tool signatures, and catalog workflow.
- **97 pytest tests** (`test_catalog_sync.py`) covering JSON schema, sync behavior, SQLite schema, server loading, alias lookups (40 parametrized), SKU tiers, deprecated services, provenance, and full coverage validation.
- **`ADDING_SERVICES.md`** — repeatable workflow guide with reusable prompt for AI-assisted service additions.
- **`COVERAGE_GAP_LOG.md`** — limitations log and gap analysis.
- **`CHANGELOG.md`** — this file.
- **`conftest.py`** / **`pyproject.toml`** — pytest configuration.

### Changed

- **`azure_pricing_server.py`** — refactored to load service mappings from SQLite instead of 280 lines of hardcoded Python dicts. Added lazy-loaded properties (`service_mappings`, `deprecated_service_mappings`, `sku_tiers`), tier pricing computation (`get_tier_pricing`, `list_tier_pricing`), service discovery method, and auto-summarize logic for broad queries.
- **`__init__.py`** — graceful fallback for relative import (supports both package and direct execution/pytest).
- **`.gitignore`** — excludes `azure_services.db` (runtime artifact).
- **`README.md`** — complete rewrite with architecture diagram, MCP client configs (Copilot CLI, Claude, VS Code/Cursor), data-driven workflow, catalog management commands.
- **Service name corrections** — fixed mismatches between alias targets and actual API `serviceName` values (e.g., "Azure Cache for Redis" → "Redis Cache", "Azure Functions" → "Functions", "Azure CDN" → "Content Delivery Network").

### Fixed

- Queries without a specific SKU no longer error with "must also specify --sku" — they return a grouped SKU summary.
- Redis, Functions, and CDN aliases now resolve to correct API service names.

### Coverage

- **Before (upstream)**: ~35 unique services, hardcoded aliases in Python
- **After (this fork)**: 156/156 API services (100%), 418 aliases, 61 SKU tiers, 8 deprecated entries — all in JSON

## [1.0.0] - 2025 (upstream)

### Original by charris-msft

- Initial MCP server implementation with `azure_price_search`, `azure_price_compare`, `azure_cost_estimate`, `azure_discover_skus`, `azure_sku_discovery`, `get_customer_discount` tools.
- Hardcoded service alias mappings and deprecated service entries in Python.
- Azure Retail Prices API integration (`2023-01-01-preview`).
- Savings plan and reserved instance pricing support.
- Customer discount application (default 10%).
- SKU validation with suggestions.
